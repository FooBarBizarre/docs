## Allmänt om Eldras APIer

Användare har tillgång till alla APIer som gränssnitten har. Det betyder att om man vill kan man bygga sitt eget gränssnitt, eller integrera Eldra-funktionalitet direkt i sitt eget system. Det som inte är öppet är våra AI/LLM/Agent/Telefonsamtal-apier.

## Autentisering

Alla anrop kräver två http headers för att autentisera anropet. Content-Type bör också defineras för bästa upplevelse.

```
X-Portal-Id: '{portalId}'
Authorization: 'Bearer {token}'
Content-Type: 'application/json'
```

1. X-Portal-Id
   1. ID för portal som anropet ska hantera. Alla ID som finns tillgängliga för använder går att hämta genom anrop till /me
1. Authorization
   1. Standard http header för autentisering. Token kan skapas för en användare under inställningar i gränssnittet (https://app.eldra.ai/settings)

## Integration med affärs/planerings-system

Genom våra apier kan man bygga sin egen integration med Eldra. \
Till partners erbjuder vi specialbyggda integrationer där vi hämtar data från era APIer för att populera in vårt system. Kontakta oss på [info@eldra.ai](mailto:info@eldra.ai) för att komma igång med en integration.

### Kom igång

<AccordionGroup>
  <Accordion title="Autentisering" icon="User">
    Skapa en token för din användare under inställningar: https://app.eldra.ai/settings 

    Där kan du även se alla portal-ID som din användare har tillgång till. 

    Skicka sedan med följade headers i alla anrop:

    ```
    X-Portal-Id: '{portalId}'
    Authorization: 'Bearer {token}'
    Content-Type: 'application/json'
    ```
  </Accordion>
  <Accordion title="Upptäck våra gränssnitt" icon="magnifying-glass">
    Gå till gränssnitt för att se alla slutpunkter som kan anropas.
  </Accordion>
</AccordionGroup>
