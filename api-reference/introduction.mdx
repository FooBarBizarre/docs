## Välkommen till Eldras API-dokumentation

Genom våra öppna apier kan man bygga sin egen integration med Eldra. \
Till partners erbjuder vi specialbyggda integrationer där vi hämtar data från era APIer för att populera in vårt system. Kontakta oss på [info@eldra.ai](mailto:info@eldra.ai) för att komma igång med en integration.

## Specialbyggda integrationer

Eldra kan koppla upp sig mot vilket affärs-/planeringssystem som helst. För att komma igång behövs bara två API-gränssnitt i systemet vi ska integrera med:

<Accordion title="Hämta alla kunder">
  Eldra synkar kundregister från systemet varje natt. På så sätt har alla, och rätt kunder alltid tillgång till Eldra. \
  Vi förväntar oss en lista tillbaka med följande data, attributens namn behöver inte vara samma som skrivet nedan:

  ```
  {
  	"name": "string", // Can be firstName, lastName as well
  	"phonenumber": "string",
  	"id": "string", // Can be integer as well. Will be used when calling back into integrating system
  	"street": "string",
  	"city": "string",
  	"postalCode": "string"
  }
  ```
</Accordion>

<Accordion title="Hämta aktuellt schema">
  Eldra hämtar schemat för en kund vid start för ett samtal. Schemat bör alltid vara uppdaterat så att kunden får rätt information. 

  Vi förväntar oss att skicka in följande paramterar. Det kan även skickas in i queryparameterar i anropet:

  ```
  {
  	"startDate": Date, // start date for schedule range, e.g right now
  	"endDate": Date, // end date for schedule range, e.g in three days
  	"clientId": "string", // Or number
  }
  ```

  Vi förväntar oss en lista tillbaka med följande data, attributens namn behöver inte vara samma som nedan:

  ```
  {
  	"startDate": Date,
  	"endDate": Date
  	"type": "string" // name of task, e.g. "Hemmets skötsel"
  	"by": "string" // name of the person scheduled to do the task
  	"status": "string" // indicating if the appointment is cancelled or active
  }
  ```
</Accordion>

För bättre upplevelse för personalen kan Eldra även anropa följande API-gränssnitt

<Accordion title="Skapa arbetsanteckning">
  Eldra kan efter ett samtal mellan en klient och Eldras AI skapa en arbetsanteckning i ert system. Detta underlätter för personalen då dom kan fortsätta kolla på samma ställe dom gör idag. \
  \
  Eldra kan även skicka in arbetsantekcning på begäran under ett samtal med personal.\
  \
  Följande data bör kunnas skickas in:

  ```
  {
  	"message": "string" // the actual working note
  	"by": "string" // either Eldra, or the staff name
  	"clientId": "string" // Or number
  }
  ```
</Accordion>

<Accordion title="Hämta kunskapsbas">
  Eldra kan under samtal med personal svara på frågor om rutiner och processer hos företaget. Exempelvis vad man ska göra om ingen svarar när man plingar på, om man har fått punka, eller hur man sjukskriver sig. 

  Vi förväntar oss att kunna hämta filer, eller rå text, på ett strukturerat sätt. Det kan vara genom API-anrop, eller genom nedladdningar från URLer. 
</Accordion>

## Datastruktur

<Info>
  Denna sektion är en övergripande beskrivning av hur data struktureras i Eldras system. Den finns till för att underlätta integrationer och sammarbeten.
</Info>

Datastrukturen som behövs för att Eldra ska fungera beskrivs nedan. Det är inte all data som finns i Eldra men beskriver minsta möjliga data för att komma igång. Det kan ses som en bra startpunkt i ett integrationsarbete.

| Entitet                 | Beskrivning                                              | Behöver Entitet      |
| ----------------------- | -------------------------------------------------------- | -------------------- |
| Organisation            | Övergripande entitet där alla andra entiteter kopplas in |                      |
| Portal                  | Entitet där kunder och konton kopplas in                 | Organisation         |
| Client                  | Representerar en kund                                    | Portal               |
| Account                 | Representere en användare i systemet                     | Portal, Organisation |
| PhoneCall               | Ett samtal mellan kund eller användare och Eldra AI      | Client / Account     |
| PhoneCallTranscriptLine | Varje meddelande från kund eller användare och Eldra AI  | PhoneCall            |

## Autentisering

### Kom igång

<Accordion title="Autentisering" icon="user">
  Alla anrop kräver två http headers för att autentisera anropet. Content-Type bör också defineras för bästa upplevelse.

  ```
  X-Portal-Id: '{portalId}'
  Authorization: 'Bearer {token}'
  Content-Type: 'application/json'
  ```

  1. X-Portal-Id
     1. ID för portal som anropet ska hantera. Alla ID som finns tillgängliga för använder går att hämta genom anrop till /me, samt under inställningar (https://app.eldra.ai/settings)
  1. Authorization
     1. Standard http header för autentisering. Token kan skapas för en användare under inställningar i gränssnittet (https://app.eldra.ai/settings)
</Accordion>
